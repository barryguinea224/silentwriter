## 🧠 The Problem

In every dev team, architecture dies silently.

- A junior changes a function signature used in 47 places → **nobody notices**
- Documentation drifts away from the code → **new devs are lost**
- AI agents call outdated APIs → **silent failures in production**
- Nobody wants to be the bad guy in code review → **standards erode**

**SilentWriter is that bad guy. Automated. Neutral. Always on.**

## 🛡️ What SilentWriter Does

```
git push
  │
  ├── IBM Granite analyzes the PR
  │     ├── Breaking change? → ⛔ PR BLOCKED
  │     ├── Docs outdated?   → 📝 agents.md auto-generated
  │     └── Safe change?     → ✅ PR APPROVED
  │
  └── agents.md auto-updated
        └── AI agents stay in sync with your codebase
```

### Core Features

| Feature | Description |
|---------|-------------|
| 🛡️ **PR Guardian** | Blocks breaking changes before merge |
| 📚 **Doc Sync** | Auto-generates documentation updates |
| 🤖 **agents.md** | Keeps AI agent instructions synchronized |
| 📊 **Architecture Health** | Tracks violations across the repo |
| 🎓 **Dev Education** | Explains WHY a change is blocked |

## 🤖 What is `agents.md`?

`agents.md` is a machine-readable API description that AI coding agents (IBM Bob, Claude Code, Cursor, GitHub Copilot) read directly to understand your codebase.

**SilentWriter guarantees `agents.md` stays synchronized with your actual code after every commit.**

```markdown
# agents.md — Auto-generated by SilentWriter
> Last updated: 2026-05-09 · Commit: #a4f82c1

## Detected Change
- login() : new required parameter `apiKey`

## Impact for AI Agents
- Agents calling login() must provide apiKey
- Fallback: AUTH_KEY_INVALID if omitted

## Recommendation
→ Update prompts of any agent using this function
```

## 🏗️ Architecture

```
GitHub Pull Request
      │
      ▼
  GitHub Action (.github/workflows/silentwriter.yml)
      │
      ├── engine/run.py
      │     └── Fetches PR diff via GitHub API
      │
      ├── engine/diff_analyzer.py
      │     ├── Loads .silentwriter.yml rules
      │     └── Calls Granite (or heuristic fallback)
      │
      └── Posts verdict as PR comment
            ├── 🛡️ BLOCK → explains why + suggests fix
            ├── ⚠️ WARN  → flags non-critical issues
            └── ✅ PASS  → PR can be merged

IBM Bob Integration (.bob/)
      │
      ├── custom_modes.yaml        → silent-guardian mode definition
      └── rules-silent-guardian/  → rule documentation
```

### Stack

| Layer | Technology |
|-------|-----------|
| Engine & Demo | Python 3.11+ |
| AI Analysis | IBM Granite / WatsonX (heuristic fallback when no key) |
| IDE Integration | IBM Bob via `.bob/custom_modes.yaml` |
| CI/CD | GitHub Actions |
| Live Demo | Hugging Face Spaces (Gradio) |

## ⚙️ Configuration (`.silentwriter.yml`)

```yaml
version: 1
rules:
  - name: "Protect API contracts"
    pattern: "src/api/**"
    forbidden_changes: ["change function signature"]
    auto_block: true

  - name: "Require tests on core"
    pattern: "src/core/**"
    required_tests: true
    min_coverage: 80

  - name: "Allowed dependencies"
    allow_list: ["prisma", "fastapi", "pydantic"]
    block_list: ["mongoose", "sequelize"]

  - name: "Core domain protection"
    pattern: "domain/**"
    require_review_from: ["tech-lead"]
```

## 🚀 Getting Started

```bash
# Clone the repo
git clone https://github.com/barryguinea224/silentwriter

# Install dependencies
pip install -r requirements.txt

# Run the demo locally
python app.py

# Or run the engine directly on a PR
export GITHUB_TOKEN=your_token
export WATSONX_API_KEY=your_key      # optional — falls back to heuristic mode
export WATSONX_PROJECT_ID=your_id
python engine/run.py --repo myorg/myrepo --pr 42
```
